## 1. 概述 (Overview)

MXM (Matrix Execution Module) 是 Groq TSP 的矩阵计算引擎，采用 **weight-stationary systolic array** 架构。每个 Super Lane 包含 2 个 MXM 切片 (左右各一)，每个 MXM 含 4 个 320×320 的计算平面，总计 409,600 个 MAC 单元。MXM 是 TSP 计算密度的核心贡献者。

```
16×16 Supercell:
┌──────────────────────────────────┐

│  ┌─ w00 ─ w01 ─ ... ─ w0,15 ─┐  │
│  │  │       │           │     │  │
│  │  MAC    MAC   ...    MAC   │  │
│  │  │       │           │     │  │

│  │  │       │           │     │  │

│  │ w15,0 ─ w15,1 ─ ... ─ w15,15│ │
│  └────────────────────────────┘  │
│  输入: Activation Bus (16-wide)  │
│  输出: Partial Sum Bus (16-wide) │
└──────────────────────────────────┘

每个 MAC:
┌─────────────────────┐
│  Weight Reg (W)      │ ← 权重固定 (weight-stationary)
│  Activation Reg (A)  │ ← 激活值流过
│  Partial Sum Reg (P) │ ← 部分和累加
│  P = P + A × W       │
└─────────────────────┘
```

## 4. 数据流方式详解 (Dataflow Analysis)

### Weight-Stationary (权重固定)

MXM 采用 **weight-stationary** 数据流:

```
Step 1: 权重加载 (Weight Load)
  ┌──────────────────────────────────┐
  │  Weights 通过方向数据流进入 Plane    │
  │  W[0][0] ~ W[319][319]           │
  │  所有 4 个 plane 并行加载          │
  │  < 40 cycles 完成                 │
  └──────────────────────────────────┘

Step 2: 激活值流 (Activation Stream)
  ┌──────────────────────────────────┐
  │  方向: East → West 或 West → East  │
  │  每个 cycle 一个 320-element 向量   │
  │  激活值水平流过 systolic array     │
  └──────────────────────────────────┘

Step 3: 部分和 (Partial Sum Propagation)
  ┌──────────────────────────────────┐
  │  部分和沿垂直方向传播 (South → North)│
  │  在每行 MAC 处累加: P += A × W   │
  │  最后一行输出完整结果              │
  └──────────────────────────────────┘
```

### Wavefront 数据流

TSP 的 "wavefront" 概念来自指令-数据交汇的方式:

```
时间 t:  ICU 发射 MMUL 指令
         SL 0 开始执行
         数据流从 MEM 开始向东移动

时间 t+1: SL 0 (继续), SL 1 (新指令)
          数据流移动到下一个 MXM 切片

时间 t+n: 所有 SL 正在执行
          数据流在多个 MXM 切片间传播
          
产生"波纹"效果 — 20 个 SL 的 MXM 呈波浪式执行
```

### 链式输出 (Chaining Output)

MXM 输出可以直接 **chain** 到 VXM 进行后处理:

```
MXM(矩阵乘) → SXM(重排) → VXM(激活函数, BN) → MEM(存储)
  输出流      路由         后处理               写回
```

## 5. 矩阵乘法执行流程 (Matrix Multiply Flow)

### C = A × B (矩阵乘示例)

```
A: M×K matrix, B: K×N matrix, C: M×N result

TSP 映射:
  A → 激活值流 (East-West)
  B → 权重 (Weight-Stationary, 预加载到 MXM planes)
  C → 输出流

执行步骤:
1. 权重加载: B 矩阵的列向量加载到 Plane 0-3
2. 激活值输入: A 矩阵的行向量从 MEM 流入
3. Systolic 计算:
   - 每个 cycle, 激活值向右传播一列 MAC
   - 部分和向下传播
   - 经过 320 cycles, 所有部分和计算完成
4. 结果输出: C 矩阵的结果向量流出到 SXM/VXM

大矩阵 (M,N > 320) 的分解:
  - 编译器将大矩阵分片为 320×320 的块
  - 循环利用 MXM planes
  - 数据流调度确保连续性
```

## 6. 4 Plane 使用策略 (4-Plane Usage)

4 个 320×320 plane 的典型使用模式:

### Pattern 1: 大矩阵乘法

```
单次: 4 × (320×320) 独立矩阵乘法
应用: 多头注意力中的 4 个 head 并行
```

### Pattern 2: FP16 精度提升

```
FP16 实现:
  Plane 0: A_low × B_low (低字节)
  Plane 1: A_low × B_high
  Plane 2: A_high × B_low  
  Plane 3: A_high × B_high
  组合: 4 个部分和 → FP32 累加
```

### Pattern 3: 批量处理

```
同一权重 (权重固定), 4 组不同激活值:
  Plane 0: Batch 0
  Plane 1: Batch 1
  Plane 2: Batch 2
  Plane 3: Batch 3
  4× 吞吐量
```

## 7. 面积与功耗 (Area & Power)

### MXM 面积分解 (推测)

| 子模块 | 单个 MXM | 全芯片 (40 MXM) |
|--------|---------|----------------|
| MAC 阵列 (409,600 MACs) | ~12 mm² | ~480 mm² |
| 权重存储 (4×320×320) | ~3 mm² | ~120 mm² |
| 数据路径布线 | ~2 mm² | ~80 mm² |
| 控制/时序 | ~1 mm² | ~40 mm² |
| **每 MXM 总计** | **~18 mm²** | **~720 mm²** |

注意: 芯片总面积 725 mm², MXM 不可能独占 720 mm²。上述为单个 MXM 切片的理想面积估算。实际实现中 MAC 阵列会与 SRAM 和其他切片共享面积，而且 409,600 MACs 在 14nm 的实际面积远小于上述估算。

**修正估算** (14nm):
- 每个 16×16 Supercell MAC: ~0.01 mm²
- 每个 Plane (400 SCells): ~4 mm²
- 4 Planes: ~16 mm²
- 40 MXM 全芯片: ~640 mm² — 不现实
- **实际**: 每个 MXM 约 8-12 mm², 全芯片 MXM 总面积约 320-480 mm²

### 每 MXM 的 14nm MAC 面积估算

| 精度 | MAC 类型 | 每 MAC 面积 | 409,600 MACs 面积 |
|------|---------|-----------|------------------|
| INT8 | 8×8-bit mul + 16-bit add | ~200 μm² | ~82 mm² |
| FP16 | 16-bit FP mul + add | ~800 μm² | ~328 mm² |
| FP32 | 32-bit FP mul + add | ~3,200 μm² | ~1,310 mm² |

实际 MXM 使用混合精度 MAC (INT8/FP16)，面积在 100-300 mm² 范围，合理估算约 30-50% 芯片面积用于 MXM。
