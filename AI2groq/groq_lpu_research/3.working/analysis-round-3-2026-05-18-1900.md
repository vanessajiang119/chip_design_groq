## 1. Groq LPU 核心架构概要

### 1.1 设计哲学：确定性（Determinism）

Groq LPU 最根本的设计原则是 **确定性计算**。与 GPU 不同，LPU 在硬件上**完全去除了所有反应式元件**（reactive elements）：

| 去除的硬件单元 | GPU 的问题 | Groq 的方案 |
|---|---|---|
| Cache（缓存） | 未命中延迟不可预测 | 全部使用 SRAM，编译器静态分配地址 |
| Out-of-Order Execution（乱序执行） | 动态调度引入 jitter | 编译器静态调度，cycle-accurate |
| Branch Predictor（分支预测） | 预测失败 penalty | 无分支，全部 predicated execution |
| Arbiters / Routers（仲裁器） | 动态争用引入不确定性 | 编译器预计算所有路由路径 |
| Speculative Execution（投机执行） | 无效指令浪费功耗 | 编译器只发射已确认需要的指令|

> **核心结果**：编译器在编译时就精确知道**每一个 instruction 在哪个 cycle 到达哪个功能单元**，芯片上 0% 的功耗浪费在硬件调度逻辑上。

### 1.2 功能切片架构（Functional Slicing）

TSP（Tensor Streaming Processor）采用**垂直功能切片**组织，而非传统的多核平铺：

```
         ┌──────────────────────────────────────────┐
         │          Instruction Control             │  ← ICU (指令控制)
         │   ICU   │  MEM  │  VXM  │  MXM  │  SXM  │
    ──►  │   ──►   │  ──►  │  ──►  │  ──►  │  ──►  │  ← 数据流 East→West
         │   ICU   │  MEM  │  VXM  │  MXM  │  SXM  │
         │   ...   │  ...  │  ...  │  ...  │  ...  │  ← 20 tiles × 16 lanes
         │   ICU   │  MEM  │  VXM  │  MXM  │  SXM  │
         └──────────────────────────────────────────┘
                ↑ 指令流 North→South
```

| Slice 类型 | 全称 | 功能 |
|---|---|---|
| ICU | Instruction Control Unit | 取指与指令分发（< 3% die area）|
| MEM | Memory Slice | SRAM 读写操作 |
| VXM | Vector Execution Module | 向量算术运算 |
| MXM | Matrix Execution Module | 矩阵乘累加（MAC）|
| SXM | Shift Execution Module | 向量移位/旋转 |

### 1.3 关键硬件规格

| 参数 | 数值 |
|---|---|
| 工艺节点 | 14nm（第一代），路线图指向 4nm |
| 主频 | 900 MHz |
| Die size | 25 × 29 mm |
| 片上 SRAM | ~230 MB（全球共享，无 cache 层次）|
| SRAM 带宽 | ~80 TB/s |
| SIMD 通道 | 320 lanes（20 tiles × 16 lanes）|
| 逻辑数据流 | 每 lane 64 个 stream（32 East + 32 West）|
| 独立指令队列 | 144 个 |
| 互联拓扑 | Dragonfly，最多 10,440 TSPs |
| 片间同步 | Plesiosynchronous + HAC/SAC |

### 1.4 TruePoint 数值系统

| 精度 | 应用场景 |
|---|---|
| FP32 | Attention logits（误差敏感）|
| Block FP | MoE 权重（无 measurable 精度损失）|
| FP8 | 激活层存储（误差容忍）|
| 100-bit 累加 | 所有运算的中间累加（lossless）|
