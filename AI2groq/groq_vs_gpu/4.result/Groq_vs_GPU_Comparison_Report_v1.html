Groq TSP vs GPU 架构对比报告

Groq TSP vs NVIDIA/AMD GPU Architecture Deep Dive — 2026-05-18

1. 报告概览

本报告系统对比 Groq TSP (Tensor Streaming Processor) 与主流 GPU (NVIDIA CUDA/AMD CDNA) 在架构理念、计算单元、存储层次、编程模型、以及实际性能方面的根本差异。Groq 的 LPU (Language Processing Unit) 在 2024-2025 年间因极低的推理延迟和高能效比引发了广泛关注，甚至以约 $200 亿 的价格将其 LPU 技术授权给 NVIDIA。本报告旨在帮助芯片架构师和 AI 基础设施决策者理解两种技术路线的本质区别及各自适用场景。

报告覆盖范围包括：Groq TSP 的确定性执行模型与功能切片架构、NVIDIA GPU 的 SIMT 模型与 Tensor Core 设计、SRAM 流式存储与 HBM+CACHE 层次对比、GroqFlow 静态编译与 CUDA 动态调度对比，以及基于 MLPerf、第三方评测的客观性能数据。

2. 架构哲学对比

2.1 Groq TSP — 确定性执行 (Deterministic Execution)

Groq TSP 的设计核心理念是消除所有"反应式硬件"——即那些引入非确定性的硬件机制。这包括：无缓存（所有存储为 SRAM，访问延迟固定）、无需分支预测（两条路径均计算，通过掩码选择）、无需乱序执行（指令严格按编译器确定的顺序执行）、无动态仲裁（所有路由决策在编译时完成）、无缓存一致性协议（因为没有缓存）。编译器被授予完全控制权——在编译时静态调度每一条指令、每一次数据移动和每一个时钟周期。

关键结果：编译器在执行开始前即可知道程序的精确周期数，运行时抖动 <1 微秒。这种"软件定义硬件"的理念使得推理延迟完全可预测，服务质量保证 (SLA) 可以做到硬实时级别。

2.2 NVIDIA GPU — SIMT + 动态调度

NVIDIA GPU 采用 SIMT (Single Instruction, Multiple Threads) 模型。每个 SM (Streaming Multiprocessor) 包含数百个 CUDA Core 和 Tensor Core，以 32 线程为一组（一个 Warp）执行相同指令。硬件 Warp 调度器在运行时动态决定哪个 Warp 发射指令——当一个 Warp 因访存而停顿，调度器零成本切换到另一个就绪的 Warp。这种以大量并发线程隐藏延迟的设计是 GPU 的核心哲学。

GPU 的复杂性体现在：动态 Warp 调度器、多级缓存层次（L1/L2）、缓存一致性协议、原子操作支持、以及处理分支发散 (warp divergence) 的机制。这种复杂性换取了极大的灵活性——GPU 可以高效处理任意计算模式（训练、推理、图形、科学计算），但也引入了显著的延迟不确定性。

3. 计算单元对比

3.1 Groq TSP 功能切片

Groq TSP 将芯片按功能类型分解为垂直列（称为 slices）。每片晶粒包含 20 个 Super Lane，每个 Super Lane 包含一组功能切片管线：ICU（取指译码分发，占芯片面积 <3%）、MEM（SRAM 读写，每 Super Lane 2 个，共 88 个独立 bank，220 MB 总容量，80 TB/s 带宽）、VXM（向量运算，ALU/激活函数）、MXM（矩阵乘累加，共 409,600 MAC 单元）、SXM（移位/旋转/路由/C2C I/O）。数据流为二维正交：指令垂直流动（北→南），数据水平流动（东→西），在交汇点执行计算。

3.2 NVIDIA GPU 计算单元

NVIDIA SM 包含 CUDA Core（标量 ALU/FPU，H100 有 16,896 个）和 Tensor Core（矩阵乘累加专用单元，H100 有 528 个第四代 Tensor Core）。CUDA Core 自 G80（2006）起存在，采用 SIMT 模型执行通用并行计算。Tensor Core 自 Volta（2017）起引入，操作矩阵 tile（4×4／8×8／16×16），承担现代深度学习的绝大部分计算量——H100 的 FP8 Tensor Core 算力达到 2,000 TFLOPS，而 CUDA Core 仅约 66 TFLOPS。Blackwell B200 进一步支持 FP4/FP6，峰值可达 9 PFLOPS。

4. 存储层次对比

4.1 Groq SRAM 流式存储

Groq TSP 采用纯 SRAM 片上存储设计，完全无 DRAM 或 HBM。220 MB 全局共享 SRAM 通过 88 个独立 bank 提供高达 80 TB/s 的片上带宽——比 H100 的 HBM 带宽（3.35 TB/s）高出约 24 倍。数据以流（Stream）形式在功能切片间传递：MEM → VXM/MXM/SXM 直接推送，无需寄存器文件中转。编译器精确调度每个数据元素到达每个计算单元的时间。

这种设计的代价是：单芯片最多只能容纳约 230 MB 参数。运行 Llama 2 70B（INT8 量化后约 70 GB）需要 305-572 个 LPU，功耗约 106 kW，硬件成本是同性能 H100 集群的约 40 倍。Groq 的核心假设是：对于延迟敏感的推理场景，片上 SRAM 的高带宽优势远远抵消了多芯片扩展的代价。

4.2 GPU HBM + 缓存层次

NVIDIA GPU 使用多级存储层次：HBM3/HBM3e（80-192 GB，3-8 TB/s 带宽）提供大容量片外存储；L2 Cache（40-96 MB）减少对 HBM 的平均访问次数；Shared Memory / L1（每 SM 128-256 KB）为线程块内提供低延迟数据共享；Register File（每 SM 64K-256K 32 位寄存器）保存线程上下文。GPU 通过大量并发线程（H100 最大 53,248 线程）隐藏 HBM 访问延迟。

6. 编程模型对比

6.1 GroqFlow / MLIR 静态编译

Groq 的软件栈以 GroqFlow 为核心——一个基于 MLIR 的编译器，接受 PyTorch、Keras 或 ONNX 模型作为输入，经过图捕获、MLIR 重写、静态张量放置、周期级调度等阶段，最终生成可在 TSP 上直接执行的二进制代码。编译器控制 144 条独立指令队列，精确决定每条指令在哪个周期发射到哪个功能单元。这种编译方式需要求解复杂的 ILP/CSP（整数线性规划/约束满足）问题，但在推理延迟方面提供了无与伦比的确定性。

6.2 CUDA / Triton 动态调度

NVIDIA 的编程模型以 CUDA 为核心——采用 Grid → Block → Warp → Thread 的四级线程层次。GigaThread Engine 将 Thread Block 分发到 SM，Block Scheduler 管理每个 SM 上的线程块，Warp Scheduler 每周期选择一个就绪 Warp 发射。这种三级调度机制使得 GPU 能够自适应地处理任意计算负载，但也引入了显著的调度不确定性。Triton 提供了类 Python 的更高抽象层级，能够达到 cuBLAS 约 90-96% 的性能。

7. 编译器哲学对比

两种路线的核心分歧在于"复杂度守恒"的分配方式：

  • Groq 路线：编译器承担极高复杂度（求解 ILP/CSP），硬件保持极简（无调度器/无仲裁器）。收益是运行时完全确定性，代价是编译器灵活度有限。
  • GPU 路线：编译器复杂度较低（生成 PTX/ISA），硬件承担动态调度负担（复杂 Warp Scheduler 和缓存控制器）。收益是任意负载都能运行，代价是延迟不可预测。

值得注意的收敛趋势：GPU 侧通过 CUDA Graphs（将多内核执行捕获为单次提交）、算子融合（FlashAttention）、Megakernel 设计模式等方式引入更多确定性；Groq 侧通过编译器支持动态形状、MoE 路由、多调度表切换等方式吸收更多动态行为。NVIDIA 以 $200 亿授权 Groq LPU 技术就是这一收敛趋势的最有力证明。

10. NVIDIA-Groq 交易与技术收敛

2025 年 12 月，NVIDIA 宣布以约 $200 亿 的非排他性授权协议获得 Groq LPU 技术。该交易包括：Groq 创始人 Jonathan Ross 加入 NVIDIA 担任推理架构 SVP，约 90% 的 Groq 工程团队加入 NVIDIA，GroqCloud 继续独立运营（由新 CEO Simon Edwards 领导）。NVIDIA 随后发布了 Groq 3 LPU (LP30)——每芯片 500 MB SRAM，150 TB/s 带宽，以及 LPX 机架系统（256 LPU，总 128 GB SRAM，40 PB/s 带宽）。

这一交易被业界解读为：GPU 巨头对确定性推理架构的正式认可。它验证了 Groq 技术路线的价值——当 AI 推理成为主导工作负载时，单靠 GPU 的"暴力计算 + 延迟隐藏"范式不足以满足延迟敏感场景的需求。同时也表明两种架构正在走向融合：GPU 吸收确定性特性，TSP 增强容量和灵活性。

关键结论：Groq TSP 和 GPU 不是替代关系，而是互补关系。Groq 在延迟敏感的推理场景（尤其是 batch=1）具有数量级优势，而 GPU 在训练、大模型推理、通用计算方面不可替代。NVIDIA 的 $200 亿 Groq 技术收购标志着推理架构正从"单一 GPU 通吃"走向"GPU 训练 + TSP 推理"的分工格局。对于芯片架构师而言，理解两种路线的本质差异——确定性 vs 动态、流式 vs 缓存、编译 vs 运行时调度——是设计下一代 AI 加速器的关键。
