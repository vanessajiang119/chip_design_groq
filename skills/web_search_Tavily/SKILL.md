# Web Search Tavily Skill — 实时联网网页搜索

## 架构概览

本技能采用**双引擎分层架构**：

| 层级 | 引擎 | 适用场景 |
|---|---|---|
| **主引擎** | Tavily AI Search（高精度检索） | 常规搜索、事实查询、资讯检索 |
| **降级引擎** | WebSearch + WebFetch（通用网页搜索兜底） | Tavily 不可用 / 结果不足 / 需要全文抓取 |

> **当前环境适配**：若 Tavily API 未配置，自动以降级引擎（WebSearch + WebFetch）作为主执行引擎，执行相同的检索流程和质控标准。

---

## 调用强制触发规则

以下情况 **必须** 调用本技能联网搜索：

1. **时效性内容**：问题包含最新时间、实时数据、行业资讯、政策变动、版本更新
2. **外部知识**：涉及外网资料、未收录知识库内容、海外技术文档
3. **模型知识滞后**：模型静态知识存在 **6个月以上** 滞后性，禁止自行作答
4. **专业领域查询**：产品参数、赛事行情、学术资料、EDA 工具文档
5. **海外信息**：境外技术博客、GitHub 项目、国际标准、英文技术资料

## 禁止调用场景

1. 纯创意写作、情绪对话、主观感悟、无事实依据的闲聊
2. 本地私有数据、内部涉密信息、不可公开的网络内容
3. 简短常识、固定公理、内置知识库已有明确答案的内容

## 检索执行流程

### Step 1：搜索关键词优化
- 接收标准化的搜索关键词
- 剔除口语化冗余表达
- 优化检索句式，提取核心实体和关键术语
- 英文技术术语保留原文，中文语境补充中文翻译

### Step 2：发起 Tavily 检索（主引擎）
- 调用 Tavily API，配置以下参数
- 开启自动重试机制（最多 3 次，间隔梯度递增）
- 超时设置：15s

### Step 3：异常降级 / 结果补全
- 连续重试失败（接口超时、限流、服务异常）
- 有效结果不足 2 条
- 满足任一条件 → 自动切换 **WebSearch + WebFetch** 通用搜索补全

### Step 4：多引擎结果融合
- URL 精准去重
- 标题语义相似度去重
- 低质站点过滤（营销软文、水帖、垃圾资讯、广告页面）

### Step 5：内容提纯与结构化
- 抓取页面有效内容，过滤广告、导航、页脚等冗余
- 提炼核心有效信息
- 保留原始来源链接
- 标记内容时效（发布日期 / 更新时间）
- 评估来源权重（官方 > 权威媒体 > 行业头部站点 > 一般来源）

### Step 6：回传与作答
- 将结构化检索素材回传给大模型
- 严格依托搜索结果作答，严禁脱离素材编造事实

## 参数规范

| 参数 | 必填 | 类型 | 默认值 | 说明 |
|---|---|---|---|---|
| `search_query` | 是 | string | — | 精简精准检索词，语句通顺无歧义 |
| `search_depth` | 否 | string | `advanced` | `basic` 快速 / `advanced` 深度检索 |
| `max_results` | 否 | int | `6` | 返回结果数量（1-10） |
| `time_range` | 否 | string | `all` | `now` 今日 / `week` 本周 / `month` 本月 / `all` 全时段 |
| `need_full_content` | 否 | bool | `false` | 是否抓取全文 |

### 参数使用指引

- **search_depth**：快速概览用 `basic`，需要深度分析用 `advanced`
- **time_range**：新闻/资讯类使用 `now` 或 `week`，技术文档/学术论文使用 `all`
- **need_full_content**：需要引用具体段落/数据时设为 `true`
- **max_results**：快速问答 3-4 条，深度调研 6-10 条

---

## 模型作答约束

1. **来源标注**：所有数据、事件、观点必须标注对应来源链接
2. **禁止编造**：严格依托搜索结果作答，严禁脱离素材编造事实
3. **权威优先**：多源信息冲突时，优先采信：
   - 官方平台（厂商官网、GitHub 官方仓库、标准组织）
   - 权威媒体（行业头部媒体、学术期刊、专利数据库）
   - 头部行业站点
4. **如实报告**：检索内容不足时如实说明，不强行拼凑答案
5. **时效优先**：时效性内容优先选用近期检索结果，舍弃老旧过期信息
6. **双语产出**：技术内容保持中文描述 + 专业英文术语/缩写

## 异常话术

| 场景 | 回复模板 |
|---|---|
| 检索接口异常 | 「当前网络检索服务临时异常，暂时无法获取外网实时信息。」 |
| 无相关结果 | 「暂未检索到公开权威相关资料，相关内容仅供参考。」 |
| 额度耗尽 | 「联网搜索次数已用尽，停止外部检索。」 |

---

## 内置能力

- **错误重试**：接口超时、限流、服务异常自动重试 3 次，间隔梯度递增
- **多层去重**：URL 精准去重 + 标题语义相似度去重
- **内容提纯**：自动过滤营销软文、水帖、垃圾资讯
- **来源溯源**：所有结果强制保留原始 URL，作答必须标注引用
- **深度切换**：`basic` 快速检索 / `advanced` 深度全面检索双模式
- **时效筛选**：支持限定今日 / 本周 / 本月 / 全时段内容筛选

## 使用示例

### 示例 1：实时行业资讯

**输入**：`search_query="TSMC 2026 先进封装产能 扩张计划", search_depth="advanced", time_range="month", max_results=5`

**流程**：
1. 优化关键词 → `TSMC 2026 advanced packaging capacity expansion plan`
2. Tavily 检索 → 获取行业新闻和分析报告
3. 降级备用 → 若结果不足，WebSearch + WebFetch 补全
4. 融合去重 → 合并结果，去重排序
5. 提纯 → 提取产能数据、时间节点、技术节点
6. 作答 → 带来源链接的结构化报告

### 示例 2：技术参数查询

**输入**：`search_query="Tessent MBIST 2025 新功能", search_depth="advanced", time_range="year", need_full_content=true`

**流程**：
1. 优化关键词 → `Tessent MBIST 2025 new features release notes`
2. Tavily 检索 → 获取 Si2 官网 / Siemens EDA 文档
3. 全文抓取 → `need_full_content=true` 时 WebFetch 抓取详情
4. 作答 → 对照官方文档输出新功能列表和版本差异
