# Skill 定义

将以下需求，转换为一个专业芯片的skill：chip_spec_lld

# Low-Level Design Spec (LLD，底层设计规格书)

LLD 是模块/子系统的微架构实现蓝图。它回答 "这个模块怎么实现、内部信号如何连接、寄存器怎么配、状态机怎么跳转" 等实现层面问题。典型内容：

- 微架构概述与设计目标
- 模块划分与层级结构
- 内部接口定义（信号级连接、时序图）
- 寄存器描述（CSR 地址、域定义、复位值）
- FSM 设计（状态转换图、状态编码）
- 数据通路与流水线结构
- 时钟/电源域实现细节
- 时序约束与综合指引
- DFT 集成（扫描链、BIST、边界扫描）
- 面积/功耗估计
- 验证协同（覆盖率要点、corner case）

chip_spec_hld，用于生成芯片设计的上层 HLD；

# HLD 和 LLD 在设计流程中如何配合使用？

1. **HLD 先行**：架构师 / 系统工程师完成 HLD → 团队 Review → 冻结外部接口与功能范围。
2. **LLD 基于 HLD 展开**：RTL 设计师 / 微架构工程师根据 HLD 写 LLD → Review → 冻结实现细节。
3. **RTL Coding**：严格按照 LLD 写代码，HLD 是验收标准。
4. **验证**：TB / 测试用例基于 HLD 写功能覆盖点，基于 LLD 写 corner case。
5. **后续变更**：任何改动必须同时更新 HLD + LLD，保持一致性。
