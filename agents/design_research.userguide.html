1. 简介

Design Research Agent 是一个专为 半导体芯片设计 领域打造的专业研究 Agent。它采用 Orchestrator + 4 Sub-Agent 架构，能够自动完成从课题理解、多源搜索、信息分析到最终报告输出的全流程研究工作。无论你是在进行 DFT 方案选型、3D-IC 架构调研、AI Accelerator 竞品分析 还是 EDA 工具链评估，只需提供一个研究课题，Agent 就能在多次迭代搜索和分析后，输出一份结构完整、图文并茂的 HTML 研究报告。

与普通的搜索引擎或通用 LLM 问答不同，Design Research Agent 采用了 四阶段多轮迭代收敛 的研究方法：planning（规划）→ research（搜索）→ working（分析）→ workingresult（输出），research 和 working 之间可循环多轮。每一轮研究都会基于上一轮的分析结果提出更深入的搜索方向，形成"搜索→阅读→思考→再搜索"的类人研究循环。最终报告使用 html_chip_design_spec 技能以 NVIDIA 白色主题风格生成，每节都配有专业的 draw.io 芯片架构图。

适用场景： 芯片方案技术选型 · 竞品功能对比分析 · EDA 工具评估 · 标准规范梳理 · 架构方案可行性研究 · 论文/专利调研

2. 快速开始

使用 Design Research Agent 只需要提供一个研究课题，即可自动启动完整的研究流程。

1

提交研究需求

在 Claude Code 中输入你的研究课题。例如："研究 3D-IC 设计中 DFT 方案的业界最佳实践，包括测试架构、EDA 工具支持和 IEEE 标准进展"。Agent 会自动创建 1.planning/ 等目录。

2

Agent 自动规划

planning 子 Agent 分析需求，生成 planning.yml（迭代控制配置，默认最多 5 轮）和搜索策略，推荐首轮搜索 URL 列表。

3

自动研究与分析迭代

research 子 Agent 到多源采集数据 → working 子 Agent 分析并决定是否继续。如需要更多信息，自动进入下一轮迭代，搜索新方向。

4

审查中间结果（可选）

每轮 working 输出的分析报告可供用户审查。如果研究方向偏移，可在此时提供新的搜索方向建议，Agent 会在下一轮迭代中采纳。

5

接收最终报告

workingresult 子 Agent 生成最终 HTML 报告 + .drawio 图文件。研究完成后在 4.result/ 目录下获取，可直接用于团队分享或设计评审。

5. 配置说明

Design Research Agent 的核心配置通过 planning.yml 文件控制。该文件由 planning 子 Agent 在规划阶段生成，用户可以在生成后审查和修改，以影响后续研究的方向和深度。默认配置支持最多 5 轮 迭代，搜索来源包括 IEEE Xplore、GitHub、Web 搜索和厂商文档。

配置项类型默认值说明
topicstring—研究课题描述，由用户输入
max_iterationsint5最大研究迭代轮次（1-10）
search_sourcesarray[ieee, github, web]数据来源列表，可增删
first_round_urlsarray[]首轮搜索种子地址
# 自定义配置示例 — 减少迭代并增加搜索来源 topic: "GDDR7 vs HBM4 接口标准对比" max_iterations: 3 search_sources: - ieee_xplore - github - web_search first_round_urls: - https://ieeexplore.ieee.org/search/gddr7 - https://github.com/topics/hbm-controller
建议： 对于范围明确、资料集中的课题，可以将 max_iterations 设为 3；对于探索性、跨领域的研究课题，建议使用 5-8 轮以获得更全面的分析。

6. 最佳实践

✅ 推荐做法
  • 明确指定研究范围和关注重点
  • 提供具体的关注维度（性能/面积/功耗/成本）
  • 在 Agent 请求审查时及时反馈调整方向
  • 对输出报告中的 draw.io 图进行二次编辑优化
  • 将高价值的研究报告存档到知识库
❌ 避免做法
  • 提交过于模糊的研究课题（如"研究芯片设计"）
  • 期望 Agent 访问付费/墙后内容（需手动提供 PDF）
  • 同时在多个线程中启动多个 research agent
  • 在迭代过程中频繁修改 topic 方向
研究质量提升技巧

通过以下几个简单技巧可以显著提升 Design Research Agent 的研究质量：首先，在提交课题时附上 1-2 篇相关的参考文献或背景资料，可以帮助 Agent 更准确地理解研究语境和术语。其次，如果对某些搜索来源有特别偏好或排除需求，在输入中明确说明（例如"优先参考 IEEE 论文和 JEDEC 标准，排除商业博客文章"）。最后，在 Agent 完成 2-3 轮迭代后主动审查中间分析报告，此时如果发现遗漏的重要方向，可以及时引导 Agent 调整搜索策略。

交互式调整（在迭代过程中给 Agent 反馈）比一次性提供完整需求能获得更高质量的研究输出。这是因为随着研究的深入，用户自己对问题的理解也会不断深化，二者形成正向循环。

子 Agent 架构： Design Research Agent v2.0 由 4 个独立子 Agent 组成 — planning（规划）、research（研究）、working（分析）、workingresult（输出）。配置文件位于 agents/config/ 目录，各子 Agent 可独立维护和升级。
