1.2 一句话摘要 (One-Sentence Summary)

SPI2AXI Bridge 是一个将 SPI 从设备接口 (SPI Slave) 转换为 AXI4-Lite 主设备接口 (AXI4-Lite Master) 的桥接 IP 核（Intellectual Property core），允许外部 SPI 主机（External SPI Master）通过低引脚数串行接口（Low-pin-count Serial Interface）访问 SoC 内部的 AXI 总线映射存储器和外设配置空间（AXI bus-mapped memory and peripheral configuration space）。该桥接器通过 dual-clock 异步 FIFO（Asynchronous FIFO）实现 SPI 时钟域到 AXI 时钟域的可靠跨时钟域数据传输（Clock Domain Crossing, CDC），同时支持 Standard SPI（1-wire）和 Quad SPI / QSPI（4-wire）两种工作模式，为 SoC 系统调试、固件更新和寄存器配置提供统一的串行访问入口。

1.3 功能目标 (Functional Targets)

  • 支持标准 SPI（1-wire）和 Quad SPI / QSPI（4-wire）两种工作模式，通过 CSR（Control and Status Register）可配置切换
  • 将 SPI 接收到的操作码（8-bit Opcode）和地址（32-bit Address）解码并转换为 AXI4-Lite 读写事务（Read/Write Transaction）
  • 通过 dual-clock FIFO 实现 SPI 时钟域到 AXI 时钟域的可靠跨时钟域数据传输（CDC）
  • 支持可配置的地址 Wrap 功能，在 AXI4-Lite 单次传输约束下模拟循环地址访问（Circular Address Access）
  • 提供寄存器文件（CSR Register File）用于工作模式配置、状态查询和参数配置

2.2 SPI 从设备接口 (SPI Slave Interface)

SPI 从设备接口通过 spi_sclk 时钟同步工作，支持 SPI Mode 0（CPOL=0, CPHA=0）—— sclk 空闲为低电平，数据在上升沿采样（Data Sampling on Rising Edge），数据在下降沿切换（Data Toggle on Falling Edge）。SPI 通信帧格式采用 MSB first（Most Significant Bit first）的数据顺序，支持八位操作码（Opcode）加三十二位地址（Address）的命令结构。

SPI 帧协议支持四种事务类型：内存写操作（Memory Write）帧格式为 Opcode（8-bit）+ Address（32-bit）+ Write Data（32-bit），用于向 SoC 内部存储器或外设写入数据；内存读操作（Memory Read）帧格式为 Opcode（8-bit）+ Address（32-bit）+ Dummy Cycles（可编程等待周期）+ Read Data（32-bit），用于从目标地址读取数据；寄存器写操作（Register Write）直接对 CSR 寄存器进行写访问；寄存器读操作（Register Read）无地址阶段，直接访问 CSR 寄存器空间。Quad SPI 模式下，四根数据线 sdi[3:0] 和 sdo[3:0] 同时在每个 sclk 周期传输四位数据，实现四倍吞吐。

2.3 AXI4-Lite 主设备接口 (AXI4-Lite Master Interface)

AXI4-Lite 主设备接口实现了 ARM AMBA AXI4-Lite 协议规范的轻量级主接口，数据位宽固定为 32-bit，突发长度（Burst Length）固定为 1（AxLEN = 0），仅支持单次传输（Single Transfer）。该接口包含五个独立的通道：写地址通道（AW Channel）、写数据通道（W Channel）、写响应通道（B Channel）、读地址通道（AR Channel）和读数据通道（R Channel）。每个通道使用 VALID/READY 握手协议进行流控，支持乱序传输（Out-of-Order Transaction）的 ID 管理，ID 宽度为 3-bit。读保护和写保护类型（AxPROT）固定为默认值，支持非安全访问。

第3章 顶层架构框图 (Architecture Block Diagram)

3.1 系统架构 (System Architecture)

SPI2AXI Bridge 的总体架构采用双时钟域设计，由一条跨时钟域边界（CDC Boundary）将整个设计划分为两个独立的时钟域：SPI 时钟域（SPI Clock Domain，同步于 spi_sclk）和 AXI 时钟域（AXI Clock Domain，同步于 axi_aclk）。SPI 时钟域包含 spi_slave_if（SPI 串行接口收发模块）、cmd_decoder（命令解码器）、fsm_ctrl（主状态机控制器）和 csr_regfile（配置寄存器文件）；AXI 时钟域包含 axi_master_if（AXI4-Lite 主接口控制器）和 wrap_addr_gen（地址环绕发生器）。两个时钟域之间通过三组独立的 dual-clock 异步 FIFO（Asynchronous FIFO）实现数据传输：cmd_fifo 传输命令和地址信息，wdata_fifo 传输写数据，rdata_fifo 传输读数据，实现 SPI 和 AXI 之间的可靠跨时钟域通信。

第4章 数据流与控制流 (Data Flow & Control Flow)

4.1 数据流路径 (Data Flow Paths)

写操作数据流从 SPI 引脚接收串行数据开始，经过 spi_slave_if 的串并转换（Serial-to-Parallel Conversion）后，命令和地址写入 cmd_fifo，写数据写入 wdata_fifo。两个 FIFO 异步同步到 AXI 时钟域后，axi_master_if 从 FIFO 中读取命令和写数据，发起 AXI4-Lite 写事务（AW → W → B 三通道握手）。读操作数据流则通过 cmd_fifo 将读命令同步到 AXI 域，axi_master_if 发起 AXI 读事务（AR → R），读数据写入 rdata_fifo 并同步回 SPI 域，最后由 spi_slave_if 通过并串转换（Parallel-to-Serial Conversion）将数据发送回 SPI 主机。

4.2 控制流 — FSM 状态机 (Control Flow State Machine)

主状态机（Main FSM）位于 SPI 时钟域，管理从 SPI 命令接收到 AXI 事务完成的完整控制流程。FSM 包含八个状态：IDLE（空闲等待）、OPCODE（接收操作码）、ADDR（接收地址）、DUMMY（读操作虚拟等待）、DATA（数据传输）、AXI_WR（AXI 写事务执行）、AXI_RD（AXI 读事务执行）和 DONE（事务完成）。FSM 采用 Moore 型状态机设计，输出仅取决于当前状态，不依赖输入条件，确保状态转换的确定性和可预测性。SPI 片选异常释放时，FSM 通过红色虚线路径（Abort Path）从任何非 IDLE 状态直接回到 IDLE，确保错误恢复的快速响应。

第6章 时钟、复位与电源架构 (Clock/Reset/Power)

6.1 时钟域架构 (Clock Domain Architecture)

SPI2AXI Bridge 包含两个异步时钟域：SPI 时钟域（spi_sclk）最高工作频率为 50 MHz，由外部 SPI 主机提供，为 SPI 串行接口逻辑提供时钟同步；AXI 时钟域（axi_aclk）由 SoC PLL 提供，工作频率待定（典型范围 50~200 MHz），为 AXI 主接口和地址环绕逻辑提供时钟基准。两个时钟域之间通过三组独立的 dual-clock 异步 FIFO 实现数据交换，FIFO 使用格雷码指针（Gray Code Pointer）进行跨时钟域同步，空/满标志（Empty/Full Flag）经过双级同步器（Two-Flop Synchronizer）处理，确保 CDC 数据的完整性和可靠性。

复位信号类型域描述

两个时钟域使用独立的异步复位信号，每个复位信号在其对应的时钟域内使用异步复位同步释放（Asynchronous Assertion, Synchronous De-assertion）的复位同步器结构，确保复位释放时与时钟的同步性。CDC FIFO 需要在两个时钟域都完成复位后才能正确初始化，FIFO 的复位完成状态通过跨时钟域握手信号通知另一侧。

第8章 应用场景 (Application Scenarios)

用例 1: 系统调试和配置接口

外部调试主机通过 SPI 连接 SoC 的 SPI2AXI 接口，单次 4 字节（32-bit）寄存器读写，实现对 SoC 配置空间的访问。SPI2AXI 解析命令并转换为 AXI 事务，读数据通过 SPI 返回给外部主机。此场景要求低延迟和可靠传输。

用例 2: 嵌入式系统固件更新

外部主机发送固件更新命令和数据（KB~MB 级），配置 SPI2AXI 为 QSPI 模式（4-wire）并启用 Wrap 功能。通过连续发送写命令和目标地址，利用地址自动递增实现固件数据的连续写入，要求高吞吐和地址连续递增能力。

用例 3: 芯片测试与验证

测试台通过 SPI 访问芯片内部状态，发送读命令和地址，等待 dummy cycles 后读取返回数据，验证内部寄存器和存储器值。此场景要求寄存器访问快速可靠，Wrap 功能方便连续地址扫描。

第10章 设计决策记录 (ADR)

#决策选择理由
001协议选择AXI4-Lite 而非 AXI4-Full目标应用为配置空间访问，无需 burst 等高级特性，面积更小
002CDC 实现Dual-clock FIFO数据量大需缓冲，FIFO 提供可靠的数据路径隔离
003FIFO 架构三级独立 FIFO (cmd/wdata/rdata)读/写数据路径分离，减少 AXI 侧仲裁延迟
004FSM 位置SPI 时钟域实现SPI 命令解析在 SPI 域完成，避免频繁 CDC
005QSPI 支持可选特性QSPI 提升 4 倍吞吐，适合高带宽场景
