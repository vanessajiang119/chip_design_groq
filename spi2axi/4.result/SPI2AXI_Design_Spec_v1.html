1. 模块概述 (Module Overview)

SPI2AXI IP 是一个将 SPI 从设备接口（SPI Slave）转换为 AXI 主设备接口（AXI4-Lite Master）的桥接模块。其核心设计目标是通过 SPI 总线实现对 SoC 内部 AXI 总线上的存储器和外设进行配置访问，从而在不依赖 SoC 内部 CPU 干预的前提下，完成芯片的功能配置、状态观测、寄存器读写等调试与初始化操作。由于 SPI 是一种通用且低引脚数的数字接口标准，SPI2AXI 桥接器特别适用于桌面 PC 环境下的 pattern 调试、实验室 bringup 以及芯片量产测试等场景。

该 IP 支持标准 SPI（1-wire）和 Quad SPI（4-wire, QSPI）两种工作模式，最高 SPI 时钟频率可达 50MHz。在 QSPI 模式下，4 条数据线同时传输，理论吞吐量可达标准 SPI 模式的 4 倍，满足中等带宽的配置访问需求。AXI 侧采用 AXI4-Lite 协议，提供 5 个独立通道（AW、W、B、AR、R），支持单次传输（AxLEN=0, AxSIZE=2 = 4 bytes），地址位宽 32-bit，数据位宽 32-bit。

1.2 应用场景

  • 系统调试和配置接口：通过 SPI 接口访问 SoC 内部配置空间，无需 CPU 参与
  • 低引脚数系统总线扩展：SPI 仅需 4-6 根信号线即可扩展出 AXI 总线访问能力
  • 嵌入式系统固件更新：通过 SPI 从外部加载固件到 SoC 内存空间
  • 芯片测试和验证接口：实验室 bringup 和 ATE 测试场景下的调试访问通道

3. 子模块划分 (Sub-Module Partition)

SPI2AXI Bridge IP 内部划分为 6 个功能子模块，各模块在各自时钟域内独立工作，通过预定义的内部接口进行数据和控制信号交换。以下是各子模块的职责和关键设计考量。

子模块类型时钟域职责

cmd_decoderCtrl FSMSPI (50MHz)8-bit opcode 解码，产生 AXI 传输控制信号，FSM 状态迁移

4. 状态机设计 (FSM Design)

命令解码器 (cmd_decoder) 是 SPI2AXI 的核心控制单元，采用有限状态机实现。FSM 在 SPI 时钟域内运行，响应 spi_cs_n 和 spi_sclk 信号，完成 SPI 命令帧的解析和控制信号的生成。FSM 共包含 6 个主要状态，采用 3-bit 二进制编码实现。

4.1 状态编码

状态编码 (3-bit)描述
IDLE3'b000空闲状态，等待 spi_cs_n 拉低，所有输出清零
OPCODE3'b001接收 8-bit 操作码，移位寄存器逐位采样
ADDR3'b010接收 32-bit 地址（MSB first），仅内存访问需要
DUMMY3'b011插入可编程 dummy cycles（DUMMY_CYCLES+1），等待 AXI 读返回
DATA3'b100数据传输阶段，32-bit 数据收发（MSB first）
RESPONSE3'b101AXI 写响应处理或读数据输出完成，准备返回 IDLE

4.2 状态转移图

stateDiagram-v2
    [*] --> IDLE
    IDLE --> OPCODE: spi_cs_n == 0
    OPCODE --> OPCODE: bit_count < 8
    OPCODE --> ADDR: bit_count == 8\n&& opcode==MEM_ACCESS
    OPCODE --> DATA: bit_count == 8\n&& opcode==REG_ACCESS
    ADDR --> ADDR: bit_count < 32
    ADDR --> DUMMY: bit_count == 32\n&& opcode==READ
    ADDR --> DATA: bit_count == 32\n&& opcode==WRITE
    DUMMY --> DUMMY: dummy_cnt < cfg
    DUMMY --> DATA: dummy_cnt == cfg
    DATA --> DATA: bit_count < 32
    DATA --> RESPONSE: bit_count == 32
    RESPONSE --> IDLE: axi_done || spi_cs_n == 1
    IDLE --> IDLE: spi_cs_n == 1
    OPCODE --> IDLE: spi_cs_n == 1 (abort)
    DATA --> IDLE: spi_cs_n == 1 (abort)

5. 流水线设计 (Pipeline Design)

SPI2AXI 的 SPI 命令到 AXI 事务的转换采用流水线架构，划分为 5 个阶段。由于 SPI 和 AXI 工作在不同时钟域，阶段间通过异步 FIFO 和握手信号进行数据传递。以下是完整的 SPI 写操作和读操作序列。

5.1 写操作序列

flowchart LR
    A[SPI Master] -->|SPI Frame| B[SPI Slave]
    B -->|opcode+addr+data| C[cmd_decoder]
    C -->|ctrl signals| D[async_fifo]
    D -->|CDC| E[axi_master]
    E -->|AXI AW| F[AXI Slave]
    E -->|AXI W| F
    F -->|AXI B| E
    E -->|response| D
    D -->|status| C
    C -->|done| B
    B -->|MISO| A
阶段时钟域操作延迟
1. SPI 命令接收SPIspi_sdi 串行输入，移位寄存器逐位采样8~72 SCLK
2. 命令解码SPI解析 opcode，生成 AXI 控制字（读/写/寄存器）1 SCLK
3. CDC 同步跨时钟域写数据/命令→异步 FIFO; 读请求→脉冲同步器2~N AXI clk
4. AXI 总线事务AXIAXI AW/W/B 或 AR/R 通道完成传输取决于 AXI Slave
5. 数据返回SPI读数据经 RX FIFO → 并串转换 → spi_sdo1~2 SCLK

6. 数据通路 (Datapath)

SPI2AXI 的数据通路分为写数据通路和读数据通路两条独立的路径，通过异步 FIFO 实现跨时钟域数据交换。SPI 侧数据在 spi_sclk 时钟域内完成串并转换，AXI 侧数据在 axi_aclk 时钟域内完成总线协议处理。

8. 时钟与复位 (Clock & Reset)

SPI2AXI IP 包含两个完全异步的时钟域，通过 dual-clock 异步 FIFO 和脉冲同步器实现可靠的跨时钟域数据传输。

8.4 复位方案

SPI2AXI 采用异步复位同步释放的复位同步器方案。每个时钟域有独立的复位同步器，确保复位释放时刻满足 CDC 要求。spi_cs_n 拉高（传输结束）也作为软复位信号，将 FSM 强制回到 IDLE 状态。

// 异步复位同步释放 RTL 模板
always_ff @(posedge clk, negedge rst_n) begin
    if (!rst_n) begin
        rst_sync_ff1 <= 1'b0;
        rst_sync_ff2 <= 1'b0;
    end else begin
        rst_sync_ff1 <= 1'b1;
        rst_sync_ff2 <= rst_sync_ff1;
    end
end
// rst_sync_ff2 作为该时钟域的同步复位信号

10. 地址环绕 (Address Wrap)

SPI2AXI 在 AXI4-Lite 协议基础上引入了可配置的地址环绕 (Wrap) 功能。由于 AXI4-Lite 仅支持单次传输 (AxLEN=0, AxSIZE=2 => 4 bytes)，Wrap 功能在软件层面模拟了循环缓冲区访问模式，适用于 FIFO 状的数据结构或循环配置空间访问。

10.1 Wrap 工作原理

Wrap 配置行为应用场景
Wrap = 0无环绕，地址持续递增（标准 AXI 行为）普通内存访问
Wrap = N (N > 0)环绕窗口 = N × 4 字节，地址在窗口内循环循环缓冲区、FIFO 访问

10.2 Wrap 地址序列示例

Wrap=2, 起始地址 'h100 (4-byte 对齐):

Burst 0: Write to 'h100  (第 1 个 word)
Burst 1: Write to 'h104  (第 2 个 word)
Burst 2: Write to 'h100  (回绕，重新从第 1 个 word 开始)
Burst 3: Write to 'h104
Burst 4: Write to 'h100
...

12. DFT 设计 (DFT Design)

待补充 — SPI2AXI 的 DFT 方案需根据 SoC 级 DFT 规划确定。以下为建议方案框架。

12.1 扫描链 (Scan Chain)

  • 所有时序单元（FF）应接入扫描链
  • 异步 FIFO 内部的格雷码指针寄存器需特殊处理
  • 测试时钟复用 SPI 时钟或 AXI 时钟

12.2 测试模式 (Test Mode)

  • test_mode 引脚控制功能/测试模式切换
  • 测试模式下所有时钟通过 MUX 切换到测试时钟
  • 异步 FIFO 在测试模式下工作于同步模式

12.3 Memory BIST

  • dual-clock 异步 FIFO 建议使用 MBIST 覆盖
  • 使用独立的 BIST 控制器和 BIST 时钟
